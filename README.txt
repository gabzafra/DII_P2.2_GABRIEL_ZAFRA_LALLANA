Para facilitar el testeo de la API he incluido en la raíz del proyecto una colección de
Postman con las peticiones que se piden en el enunciado DII_P22.postman_collection.json

La aplicación se basa en la práctica 1.2 e incluye clases para persistencia en memoria o
en un archivo xml. Se puede cambiar modificando los comentarios de las lineas 10 y 11 de 
ContactoService.java.

Como ampliación he decidido devolver además de los mensajes de error/éxito los códigos de
estado HTTP correspondientes.
